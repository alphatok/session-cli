# Requirements

This folder contains requirements documentation for the Session CLI project.
Each `.md` file represents a module or feature area and should be maintained as
the project evolves.

## Structure

| File | Area | Description |
|------|------|-------------|
| [core.md](core.md) | Core | Session grabbing, vault storage, MCP communication |

## Conventions

- Use present tense ("The system stores...") throughout.
- Tag status: `✅ done`, `🚧 in-progress`, `📋 planned`.
- Keep each requirement atomic and testable.
- Update this index when modules are added or removed.
