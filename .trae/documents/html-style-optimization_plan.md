# HTML 页面样式优化计划

## 项目调研结论

当前项目是一个 Session Manager 工具，前端页面位于 [templates/index.html](file:///d:/github/session-cli/templates/index.html)。页面采用 GitHub 深色主题风格，包含以下组件：
- MCP 连接状态栏
- 添加站点表单（输入框 + 按钮）
- 进度条
- 站点列表（卡片式）
- Toast 通知
- 详情弹窗

现有样式较为基础，缺少丰富的交互反馈和视觉层次。

## 优化目标

根据用户偏好（Apple-like 极简风格、Aurora glow 效果、hover 光效变化），优化方向：

1. **输入框**：添加 Aurora glow 效果（参考 Google Gemini），hover/focus 时光效增强
2. **按钮**：添加平滑过渡、hover 阴影、active 状态反馈
3. **卡片**：添加 hover 阴影、微妙的边框光效
4. **列表项**：添加 hover 背景、过渡动画
5. **进度条**：添加条纹动画效果
6. **Toast**：优化动画曲线、添加淡出效果
7. **状态点**：增强发光效果
8. **全局细节**：优化圆角、间距、阴影层次

## 修改文件

仅修改：`templates/index.html`（`<style>` 标签内的 CSS 样式）

## 优化步骤

### 步骤 1：增强 CSS 变量定义
- 添加阴影变量
- 添加过渡时间变量

### 步骤 2：优化输入框样式
- 添加 Aurora glow 边框效果
- hover 时光效增强
- focus 时边框颜色变化 + 光效扩散

### 步骤 3：优化按钮样式
- 添加 box-shadow
- hover 时阴影扩散、颜色加深
- active 时轻微缩放

### 步骤 4：优化卡片样式
- 添加 hover 阴影
- 边框透明度调整

### 步骤 5：优化列表项样式
- hover 背景变化
- 过渡动画

### 步骤 6：优化进度条
- 添加条纹动画
- 平滑过渡

### 步骤 7：优化 Toast 通知
- 添加淡出动画
- 优化动画曲线

### 步骤 8：优化状态指示器
- 增强发光效果

## 潜在风险

- 修改样式可能影响现有布局 → 确保不更改布局属性（width、height、position 等）
- 动画可能影响性能 → 使用 CSS3 硬件加速属性（transform）

## 验证方式

启动开发服务器，访问页面查看视觉效果和交互反馈。
